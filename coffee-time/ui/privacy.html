隐私优先设计

你的隐私，我们的底线

我们采用隐私优先的设计理念，从架构层面保护你的个人信息和咨询内容。以下是我们当前已采取的措施和持续建设的方向。

我们收集哪些数据

我们遵循最小化数据收集原则，仅收集提供服务所必需的信息：

  • • 账户信息：用户名、邮箱（用于登录和通知）
  • • 身份信息：仅导师申请时需要，用于验证职业背景
  • • 咨询记录：预约时间、会话时长、评价内容

我们不会收集：精确地理位置、设备指纹、浏览行为画像等与咨询服务无关的数据。

数据保护措施

传输加密

所有数据通过 HTTPS (TLS 1.2+) 传输，确保数据在传输过程中不被窃听。

密码安全

用户密码使用 bcrypt 算法加密存储，即使数据库泄露也无法还原原始密码。

访问控制

严格的权限管理，平台运营人员无法直接查看用户咨询内容。

数据留存策略

inactive 超过 2 年的账户数据将自动匿名化处理。

咨询内容保护

我们深知职场咨询内容的敏感性，因此采取以下措施：

  • 咨询内容仅对会话双方可见，平台不会主动分析或提取咨询内容用于商业目的
  • 导师无法查看用户与其他导师的咨询记录
  • 用户可以随时删除自己的咨询历史

说明： 当前阶段咨询内容以明文存储于服务器。我们正在规划消息加密存储方案，后续版本将实现端到端加密，届时即使数据库被访问，咨询内容也无法被读取。

身份验证与隐私

导师身份验证是平台信任体系的基础，但我们也高度重视验证过程中的隐私保护：

  • • 上传的工牌/证明文件不会永久保存——验证完成后将被删除或脱敏处理
  • • 验证结果仅以"已验证/未验证"状态展示，不会公开具体的公司或职位信息（除非导师自行选择公开）
  • • 社交账号（LinkedIn/脉脉）验证仅用于交叉验证，不会获取或存储你的社交账号内容

第三方服务说明

我们使用以下第三方服务，它们各自受其隐私政策约束：

支付服务

支付信息直接由支付服务商处理，我们不存储你的银行卡号或支付凭证。

字体与前端框架

Google Fonts 和 Tailwind CSS CDN 会记录你的 IP 地址以提供服务。我们计划后续将字体自托管以减少第三方依赖。

持续建设中的安全目标

安全是一个持续的过程。以下是我们计划在后续版本中实现的安全增强：

消息加密存储

实现咨询内容的加密存储，确保数据库层面的内容保护

规划中

端到端加密通信

实现客户端到客户端的端到端加密，平台无法读取咨询内容

规划中

安全审计与渗透测试

定期进行第三方安全审计，主动发现并修复潜在风险

规划中

数据隔离架构

身份数据库与业务数据库物理隔离，降低数据泄露风险

规划中

以上为规划方向，实际实施时间和范围可能根据业务需求调整。我们承诺在重大安全能力上线前，通过站内通知和邮件告知用户。
